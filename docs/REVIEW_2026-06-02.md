# Studio Stud — Technical Review & Resolution Plan (2026-06-02)

Static, read-only review of the post-migration repo, with a directly-executable resolution plan for
Cursor Composer 2.5. Scope: make the migration's new pieces (GitHub hosting, single-line install,
auto-update, outside-Roblox distribution) actually work, and prove the in-Studio/in-game function is
unchanged. Decisions in section B are baked in. Every claim cites `file:line`.

---

## A) VERDICT

The tool's **core in-game function is intact**: the daemon's Studio-facing surface — `/ping`,
`/manifest`, all `/capture/*`, `/live/*`, `/write/*` — is present and behaviorally unchanged
(`src/http.rs:83-339`), the plugin's new version handshake passes at protocol 1 and does **not** gate
capture (`plugin/StudioStud.plugin.lua:1841-1899`), and the channel-update fields are merged into the
ping non-destructively (`src/http.rs:776-784`). The **distribution layer, however, is broken end to
end**: the published single-line install cannot complete on a clean machine, and auto-update cannot
fire. The **single biggest blocker** is that the release `site/latest.json` served to users is a stale
hand-written file with no `setupUrl`/`channelSequence`/`signature`, so `install.ps1` throws before it
can download anything (`site/install.ps1:50-61`, `site/latest.json:1-11`) — and even if it got past
that, the GUI/silent install never acquires the daemon or plugin on a clean machine
(`setup/src/gui.rs:602-616`). Auto-update is independently dead: the ed25519 manifest signature never
verifies because the PowerShell signer and the Rust verifier canonicalize the JSON differently
(`src/setup_core/channels.rs:113-149` vs `scripts/publish-channel.ps1:76`), and the update trigger keys
off a semver that never changes across dev pushes (`src/setup_core/channels.rs:190-199`). All are
fixable without touching the proven core; the plan below sequences the fixes quick-wins-first and ends
at in-Studio parity (section H).

## B) DECISIONS LOG

| # | Question | Decision |
|---|----------|----------|
| D1 | Keep or drop manifest signing? | **Keep.** `verify_manifest_signature` skips when `signature` is absent (release = intentionally unsigned, working). Beta/dev stay signed — but the present signatures don't verify today (canonicalization mismatch), so signing is moved to one shared Rust canonicalization used by both signer and verifier so the signatures you already produce validate. All-zeros/`PLACEHOLDER` pubkey escape hatch retained. |
| D2 | Release manifest signed or unsigned? | **Unsigned**, via "skip-when-absent". Smallest diff; unblocks release with no CI signing-secret on the `main`/tag path. |
| D3 | Clean-machine fresh-install acquisition? | **Self-contained bundle (Option 1 / Option A in spirit).** Each channel ships one artifact (ZIP) containing setup + daemon + plugin (+ addons); encrypted on beta/dev. Fresh install extracts and finds siblings — no separate per-file download. |
| D4 | Doc rewrites in scope? | **Deferred.** Functional fixes first; doc/script staleness captured in an optional final phase (step G-D1). Exception: `install-local.ps1` bugs are functional and fixed in the main pass. |
| D5 | `requireAdministrator`? | **Switch to asInvoker.** All writes are HKCU + `%LOCALAPPDATA%` (user-owned); no elevation needed. |
| D6 | Daemon-side update CLI? | **Deprecate as no-ops.** Keep `apply_staged_on_boot` (load-bearing); hide `studio-stud update` and `serve --no-update` from help; leave as no-ops. |
| D7 | Update trigger | **`channelSequence`, not semver.** Required for "push to dev → my machine auto-updates" since `Cargo.toml` stays at `0.4.0` across dev pushes. |
| D8 | Channel delivery of new daemon/plugin | **Bundle into one per-channel artifact** (D3). Makes locked channels actually lock the real code and keeps the (daemon+plugin+setup) version set coherent. |
| D9 | User-vs-repo split | **Keep the three-tier split.** Per-machine: binaries, core plugin, registry/config, write token, channel state. Committed in git: `.studio-stud/policy.json` + addon-enable config. New: installer drops a `.studio-stud/.gitignore` so per-machine state (`.installed`, future base-ledger/stash/merge) never gets committed and clobbers concurrent teammates. |

## C) AS-BUILT DESIGN (current reference)

### C.1 Components
- **Daemon/CLI `studio-stud.exe`** (`src/`, workspace root crate `Cargo.toml:1-16`). Entry
  `src/main.rs:1-3` → `src/cli.rs` (clap). Subcommands: `status`, `doctor`, `ingest`, `analyze`,
  `query`, `capture`, `serve`/`daemon`, `update`, hidden `bench`/`live-*`/`write-*`, `policy`,
  `project`, `repo-map` (`src/cli.rs:43-266`).
- **Setup `studio-stud-setup.exe`** (`setup/`, crate `setup/Cargo.toml`). Entry `setup/src/main.rs`;
  subcommands `install` (GUI default / `--silent`), `uninstall`, `update [--check]`, `health`,
  `repair`, `repo-health`, `repo-repair` (`setup/src/main.rs:33-59`). GUI in `setup/src/gui.rs`.
- **Shared `setup_core`** (`src/setup_core/`): `channels` (manifest fetch/verify/anti-rollback),
  `crypto` (AES-256-CBC+HMAC artifact decrypt, DPAPI), `config` (registry `config.json`), `install`
  (lay payload, PATH shim, addons), `registry` (PlaceId→repo resolver), `health`, `channel_update`
  (24h-cached ping fields). Re-exported via `src/setup_core/mod.rs`.
- **Core Studio plugin** `plugin/StudioStud.plugin.lua` (3652 lines): read-only capture + live deltas +
  write transport + tab registry + settings. `PLUGIN_VERSION = "0.3.7"`, `PROTOCOL_VERSION = 1`,
  `MIN_DAEMON_PROTOCOL_VERSION = 1` (`plugin/StudioStud.plugin.lua:13-19`).
- **Distribution site** `site/`: `install.ps1` (+ `install-beta.ps1`/`install-dev.ps1` thin wrappers),
  `latest.json`, `index.html`.
- **CI** `.github/workflows/`: `ci.yml` (PR gate), `deploy.yml` (push→publish), `promote.yml`
  (dev→beta→main PRs).
- **Release tooling** `scripts/`: `package-release.ps1`, `publish-channel.ps1`, `build-local.ps1`,
  `install-local.ps1`, `keygen.ps1`, `configure-github.ps1`, orphaned `launcher.ps1`/`.cmd`.
- **Crypto helpers** `examples/`: `keygen.rs`, `encrypt-artifact.rs`, `sign-manifest.rs`.

### C.2 Install path (as built)
1. User runs `irm .../install.ps1 | iex` (release) or `install-beta/dev.ps1` (`site/install-beta.ps1:11-12`).
2. `install.ps1` fetches the channel manifest with dev→beta→release fallback (`site/install.ps1:24-45`).
3. Release: downloads `$manifest.setupUrl` → `%TEMP%\studio-stud-setup.exe`; beta/dev: downloads
   `$manifest.setupEncUrl`, prompts for password, PBKDF2-SHA256×200k → AES-256-CBC + HMAC verify, writes
   the exe (`site/install.ps1:50-127`).
4. Launches `studio-stud-setup.exe install` → GUI (`setup/src/main.rs:73-76`).
5. GUI `run_install` resolves daemon/plugin via `resolve_daemon_src`/`resolve_plugin_src`
   (sibling-of-exe / repo lookup), lays payload to `%LOCALAPPDATA%\Programs\StudioStud`, copies the
   plugin to `%LOCALAPPDATA%\Roblox\Plugins`, prepends `…\bin` to user PATH via `setx`, writes
   `config.json` (`setup/src/gui.rs:602-633`, `setup/src/install_flow.rs:27-65`,
   `src/setup_core/install.rs:40-138`).

### C.3 Update path (as built)
- `studio-stud-setup update [--check]` → fetch+fallback manifest → `verify_manifest_signature` →
  `check_anti_rollback` → `channel_update_available` → if not check & available, `apply_channel_update`
  (`setup/src/main.rs:121-174`).
- `apply_channel_update`: stop running daemon (admin shutdown), download `binaryUrl`+`pluginUrl` to
  temp, run headless update, save config (`setup/src/update_apply.rs:28-63`).
- Daemon `serve` only applies a previously staged `.exe.new` swap on boot (`src/update.rs:147-220`,
  `src/cli.rs:715-716`); it does **not** fetch.
- Plugin shows an "Update available" chip from `ping` fields (24h throttled)
  (`plugin/StudioStud.plugin.lua:62-94,1883-1890`), fed by `ChannelUpdateCache.ping_fields`
  (`src/setup_core/channel_update.rs:37-73`) merged in `manifest_json_with_update`
  (`src/http.rs:776-784`).

### C.4 Distribution / CI (as built)
- `deploy.yml` on push to `main`→publish `site/` to gh-pages root; on `development`→encrypt setup +
  sign + publish `site/dev/`; on `v*` tag→GitHub Release with the 3 assets (`.github/workflows/deploy.yml`).
- `package-release.ps1` builds, assembles `dist/`, and **regenerates** `site/latest.json` with
  `setupUrl` + `channelSequence` (`scripts/package-release.ps1:99-113`) — but only in the build job's
  workspace.
- Branch model dev→beta→main via `promote.yml`; protection via `configure-github.ps1`.

### C.5 In-Studio runtime (as built, unchanged by migration)
- Plugin polls `/capture/request` every ~2-3s, builds a full DataModel snapshot over ordered root
  services + curated `CLASS_PROPERTIES`, uploads via body/chunked transfer, then live deltas to
  `/studio-stud/live/delta` with a debounce (`docs/usage.md:55-75`,
  `plugin/StudioStud.plugin.lua:1741-1839`).
- Daemon materializes snapshots into one SQLite live DB per place (`syncs.db`), applies deltas in
  transactions, learns PlaceId→repo on capture complete (`src/http.rs:605-659`).
- Writes are token-gated, policy/allowlist-gated, atomic, place-id-checked (`src/http.rs:383-464`).
